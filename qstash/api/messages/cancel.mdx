---
title: "Cancel Message"
description: "Stop delivery of an existing message"
api: "DELETE /v2/messages/:messageId"
authMethod: "bearer"
---


Cancelling a message will remove it from QStash and stop it from being delivered in the future.
If a message is in flight to your API, it might be too late to cancel.

## Request

<ParamField path="messageId" type="string" required>
The id of the message to cancel.
</ParamField>



## Response

This endpoint only returns `202 OK`



<RequestExample>



```sh 
curl -XDELETE https://qstash.upstash.io/v2/messages/msg_123 \
  -H "Authorization: Bearer <token>"
```

</RequestExample>

<ResponseExample>
```text 202 Accepted
OK
```

</ResponseExample>
